# مركز تحسين مسارك للتدريب — منصة كاملة

نظام إدارة تدريب حقيقي (Full-Stack) بواجهة عربية RTL وقاعدة بيانات SQLite دائمة، يشمل:

- موقع عام: الرئيسية، صفحة الدورات مع فلاتر، صفحة تفاصيل كل دورة.
- تسجيل دخول وإنشاء حسابات فعلي (JWT + تشفير كلمات المرور بـ bcrypt).
- 3 لوحات تحكم حقيقية متصلة بقاعدة البيانات: مدير، مدرب، متدرب.
- تسجيل حقيقي في الدورات، تتبع تقدم، شهادات تصدر تلقائياً عند إكمال الدورة، إشعارات.

## البنية التقنية

- **Backend:** Node.js + Express
- **قاعدة البيانات:** SQLite عبر الوحدة المدمجة في Node (`node:sqlite`) — لا حاجة لتثبيت أو تشغيل خادم قاعدة بيانات منفصل، والبيانات تُحفظ في ملف `data.sqlite` بشكل دائم.
- **المصادقة:** JWT + bcryptjs
- **الواجهة الأمامية:** HTML/CSS/JS بدون إطار عمل، صفحة واحدة (SPA) تتصل بالـ API عبر fetch.

> ملاحظة مهمة: `node:sqlite` ميزة تجريبية في Node.js وتتطلب **Node.js نسخة 22.5 أو أحدث**. تأكد أن خدمة الاستضافة التي ستختارها تدعم هذه النسخة.

## التشغيل محلياً

```bash
cd webapp
npm install
npm start
```

سيعمل الموقع على `http://localhost:3000` (الواجهة الأمامية وواجهة الـ API يعملان من نفس الخادم).

عند أول تشغيل، يتم إنشاء قاعدة البيانات وتعبئتها تلقائياً ببيانات تجريبية (دورات، مستخدمون، تسجيلات).

### حسابات تجريبية جاهزة

| الدور | البريد الإلكتروني | كلمة المرور |
|---|---|---|
| مدير | admin@tahseenmsarak.com | admin123 |
| مدرب | mohammed@tahseenmsarak.com | instructor123 |
| متدرب | abdullah@example.com | trainee123 |

يمكن أيضاً إنشاء حسابات جديدة فعلياً من شاشة "حساب جديد" في الموقع (كمدرب أو متدرب).

## متغيرات البيئة

انسخ `.env.example` إلى `.env` وعدّل القيم قبل النشر:

```
PORT=3000
JWT_SECRET=غيّر-هذه-القيمة-لمفتاح-سري-طويل-وعشوائي
DB_PATH=./data.sqlite
```

**مهم جداً:** غيّر `JWT_SECRET` إلى قيمة عشوائية طويلة قبل نشر الموقع فعلياً على الإنترنت، وإلا يمكن لأي شخص تزوير جلسات دخول.

## النشر على استضافة حقيقية بدومين خاص

بما أن هذا مشروع Node.js عادي بقاعدة بيانات كملف واحد (SQLite)، يمكن نشره على أي منصة تدعم Node.js. الخيارات الأسهل:

### الخيار 1: Render.com (مجاني للبداية، سهل جداً)
1. ارفع مجلد `webapp` إلى مستودع GitHub.
2. أنشئ حساباً على render.com واختر "New Web Service".
3. اربط المستودع، واضبط:
   - Build Command: `npm install`
   - Start Command: `npm start`
   - أضف متغيرات البيئة (`JWT_SECRET`) من نفس الصفحة.
4. تأكد أن نسخة Node المختارة 22 أو أحدث (Render يدعم تحديد النسخة عبر ملف `.node-version` أو حقل `engines` في package.json، وهو موجود مسبقاً).
5. بعد نشر الموقع، اذهب إلى إعدادات الدومين في Render وأضف نطاقك الخاص (بعد شرائه من أي مزود نطاقات) ووجّه سجلات DNS كما يوضح Render.

### الخيار 2: Railway.app
نفس الفكرة تقريباً: ربط GitHub، ضبط متغيرات البيئة، ثم ربط الدومين من إعدادات المشروع.

### الخيار 3: VPS خاص (Hostinger, DigitalOcean, إلخ)
1. ثبّت Node.js 22+ على الخادم.
2. ارفع ملفات المشروع (باستثناء `node_modules`) وشغّل `npm install && npm start`.
3. استخدم `pm2` لإبقاء السيرفر يعمل باستمرار: `npm install -g pm2 && pm2 start server.js`.
4. اربط الدومين عبر Nginx كـ reverse proxy إلى المنفذ المحدد في `PORT`، ثم فعّل شهادة SSL مجانية عبر Certbot.

### ملاحظة عن قاعدة البيانات عند النشر
ملف `data.sqlite` يُنشأ تلقائياً بجانب الكود. على بعض المنصات المجانية (مثل الطبقة المجانية في Render) قد يُعاد ضبط القرص عند إعادة تشغيل الخدمة، مما يفقد البيانات. إذا احتجت بيانات دائمة 100% على تلك المنصات، فعّل خيار "Persistent Disk" إن وُجد، أو انتقل لاحقاً إلى قاعدة بيانات مُدارة (مثل PostgreSQL على Render) — هيكلة الكود في `db.js` مبسطة بحيث يسهل استبدالها لاحقاً.

## نقطة مهمة عن الدفع الإلكتروني

زر "شراء الآن" في هذا الإصدار يسجّل عملية شراء ويُدخل المتدرب في الدورة **مباشرة داخل قاعدة البيانات دون بوابة دفع فعلية** (الطلب يُحفظ بحالة "مدفوع" تلقائياً للتجربة). لتفعيل دفع حقيقي بأموال فعلية، تحتاج لاحقاً:
1. حساب تاجر إلكتروني لدى بوابة دفع سعودية (مثل Moyasar أو PayTabs أو Tap).
2. مفاتيح API الخاصة بالحساب.
3. عند توفرها، يمكن ربطها بسهولة داخل `routes/courses.js` (نقطة `/enroll`) بدلاً من إنشاء الطلب مباشرة.

## هيكل المشروع

```
webapp/
  server.js          نقطة الدخول
  db.js               تعريف الجداول والبيانات الأولية
  middleware/auth.js   التحقق من تسجيل الدخول والصلاحيات
  routes/
    auth.js            تسجيل الدخول / إنشاء حساب
    courses.js          الدورات (عرض، إضافة، تعديل، حذف، تسجيل، تقييم)
    admin.js             إحصائيات وبيانات لوحة المدير
    instructor.js        بيانات لوحة المدرب
    trainee.js            بيانات لوحة المتدرب
  public/index.html     الواجهة الأمامية الكاملة (SPA)
```

## أوامر API الرئيسية (للتوسعة المستقبلية)

- `POST /api/auth/register` `{name,email,password,role,phone}`
- `POST /api/auth/login` `{email,password}`
- `GET /api/courses` — كل الدورات
- `POST /api/courses` (مدير/مدرب) — إضافة دورة
- `PUT /api/courses/:id` (مدير/مدرب) — تعديل دورة
- `DELETE /api/courses/:id` (مدير) — حذف دورة
- `POST /api/courses/:id/enroll` (متدرب) — تسجيل في دورة
- `PUT /api/trainee/my-courses/:enrollmentId/progress` — تحديث نسبة التقدم (يصدر شهادة تلقائياً عند 100%)

هذه النقاط جاهزة في الخلفية حتى لو لم تُستخدم كلها في واجهة لوحة المدير الحالية، لتسهيل إضافة صفحات إدارة أعمق لاحقاً (مثل شاشة إضافة/تعديل دورات كاملة).
